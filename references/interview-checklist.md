# Dify Workflow 需求访谈清单

## 业务
- 这个工作流解决什么问题？
- 成功标准是什么？
- 目标用户是谁？
- 单轮任务还是多轮对话？

## 输入
- 文本 / 图片 / 文件 / URL / JSON / 表单？
- 是否有固定字段？
- 是否需要参数提取？

## 输出
- 纯文本 / Markdown / JSON / 卡片 / 文件 / 文档 / API 回写？
- 是否必须是结构化卡片？
- 是否要生成 Dify 可导入 YAML？

## 流程
- 核心步骤有哪些？
- 哪些分支必须存在？
- 是否需要循环收集？
- 是否需要人工兜底？
- 是否需要错误处理？

## 外部依赖
- API / 数据库 / 知识库 / Webhook / 插件 / Tool
- 鉴权方式是什么？

## 技术约束
- workflow / advanced-chat / rag_pipeline？
- 新建还是改造？
- app id / 页面 URL？
- 模型 / provider？
- 是否中文优先？
