## 3. 飞书文档 / 长介绍版

我刚发布了自己的第一个 GitHub 项目：`dify-workflow-builder`。

这个项目面向 Dify 用户和 Agent 使用者，目标不是单纯“写 DSL/YAML”，而是让 Agent 真正具备一种完整能力：

- 从业务需求出发
- 梳理目标、输入、输出、流程和依赖
- 判断适合的 Dify 模式
- 设计工作流结构
- 输出可导入的 `Dify DSL YAML file`
- 对现有 YAML 做 review / refactor / validate
- 在用户直接给 Dify URL 的情况下，识别现有 app 并继续搭建或修正

如果你平时在做 Dify workflow、chatflow、rag_pipeline 或 agent-chat，或者你经常遇到“YAML 能不能导入”“导入后为什么跑不通”“改了但页面没变化”这类问题，这个项目会比较适合你。

项目地址：
https://github.com/lif199010-stack/dify-workflow-builder
