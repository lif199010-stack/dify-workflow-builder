# Dify Workflow 需求访谈模板

## Phase 1: 业务背景（5W1H）

### What - 解决什么问题？
- [ ] 这个 workflow 的核心业务目标是什么？
- [ ] 期望达到什么效果？（如：自动回复率 80%、收集完整率 90%）

### Who - 目标用户？
- [ ] 用户角色：（内部员工/外部客户/管理员/其他）
- [ ] 用户特征：（技术水平、使用场景、期望体验）

### When - 触发时机？
- [ ] 触发方式：
  - [ ] 用户主动发起（聊天/点击）
  - [ ] 定时触发（每天/每周/特定时间）
  - [ ] 事件触发（收到消息/状态变更/外部回调）
  - [ ] 其他：______

### Where - 使用场景？
- [ ] 使用平台：（飞书/钉钉/网页/微信/其他）
- [ ] 接入方式：（Webhook/API/直接访问）

### Why - 为什么需要？
- [ ] 当前痛点：（人工成本高/响应慢/易出错/其他）
- [ ] 替代方案：（已有系统/人工流程/其他工具）

### How - 业务流程？
- [ ] 请描述完整的业务流程：
  1. 用户做什么？
  2. 系统做什么？
  3. 输出什么？
  4. 异常如何处理？
