# Promo Copy

## 中文短文案

我刚发布了自己的第一个 GitHub 项目：`dify-workflow-builder`。

这是一个专门帮助 Agent 从**业务需求一路搭到 Dify 工作流落地**的 Skill。

它不只是生成 YAML，还能：
- 梳理业务需求
- 设计 Dify 工作流
- 输出 `Dify DSL YAML file`
- review / refactor / validate 现有 YAML
- 通过 Dify URL 直接识别现有 app 并继续修正

适合想用 Agent 提升 Dify 工作流搭建效率的人。

项目地址：
https://github.com/lif199010-stack/dify-workflow-builder

## 中文长文案

我把最近这段时间沉淀的 Dify 工作流搭建经验，整理成了一个正式开源项目：
`dify-workflow-builder`

这个项目的目标不是单纯“帮你写 YAML”，而是让 Agent 具备一种完整能力：
从业务梳理、需求确认、模式判断、工作流设计，到输出 Dify DSL YAML file，再到 review / refactor / validate / 调试 / 落地，形成一套真正能推进项目的流程。

如果你平时在做：
- Dify workflow
- chatflow
- rag_pipeline
- agent-chat

或者你经常遇到：
- 不知道该选什么模式
- YAML 能不能导入不确定
- 导入后流程不通
- 改了但页面没变化
- 用户只给一个 Dify URL，希望 Agent 自己进去看并继续改

那这个项目应该会对你有帮助。

项目地址：
https://github.com/lif199010-stack/dify-workflow-builder
